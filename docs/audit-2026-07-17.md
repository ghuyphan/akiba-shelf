# Akiba Shelf — Full Codebase Audit

Date: 2026-07-17 · Scope: security, code cleanliness, project structure, performance, best practices
Method: 5 parallel audit agents + manual verification of security-critical paths. All findings verified against source with file:line evidence.

---

## 1. Security

**Overall posture: strong. No Critical or High findings.**

Verified strengths:

- **RLS coverage is complete.** All 16 tables have `enable row level security` (checked `create table` vs RLS across all 49 migrations).
- **All 61 `security definer` functions set `search_path = ''`** — no search-path hijack surface.
- **No anonymous writes to orders.** Early policies "Anyone can insert orders/order_items" (`20260709000000:59,82`) were dropped wholesale by the multi-shop migration (`20260712160000:132-136`, dynamic drop of all policies on 6 tables). Final state: orders/order_items readable only by shop staff; writes only via RPCs.
- **`create_order` RPC is server-authoritative** (`20260716063401:187-330`): advisory xact lock on `client_request_id` for idempotency, recovery token ≥ 32 chars stored as SHA-256 hash, product rows locked `FOR UPDATE` ordered by id (deadlock-safe), server-side stock validation and price/promotion computation, demo-shop rejection.
- **Edge functions are well hardened.** `invite-shop-member`: origin allow-list, caller JWT verification, owner-role check, UUID/email validation, 20 invites/hour rate limit, compensating revoke on email failure, generic error messages. `notify-new-order`: origin check, order lookup gated by recovery-token hash, dedupe via unique constraint (`23505`), dead subscription cleanup.
- **Grants are least-privileged**: sensitive functions (`find_auth_user_by_email`, `resolve_invitation_user`, `expire_pending_orders`) granted only to `service_role`; anon only gets `create_order`, `get_customer_order`, `cancel_customer_order` (all token-gated).
- **No XSS sinks**: zero `dangerouslySetInnerHTML` / `innerHTML` / `eval` in `src/`.
- **No committed secrets**: `.env.local` is gitignored (`.gitignore:7`) and contains only public `VITE_*` vars (anon key is public by design); `.env.example` matches exactly what the code reads.

Remaining items (Low):

| # | Finding | Location | Recommendation |
|---|---|---|---|
| S1 | `notify-new-order` is unauthenticated by design (token-gated) but has no rate limiting beyond dedupe; a leaked recovery token allows push spam to staff | `supabase/functions/notify-new-order/index.ts:52-96` | Optional: per-IP/order throttle |
| S2 | Recovery token is a bearer credential in the order-status URL — anyone with the link can read/cancel the order | `get_customer_order` / `cancel_customer_order` grants (`20260711120000:158`) | Acceptable for booth flow; avoid logging full URLs |
| S3 | VAPID public key hardcoded in CI workflow | `.github/workflows/validate.yml:90` | Move to a repo variable (not a secret leak — public key) |
